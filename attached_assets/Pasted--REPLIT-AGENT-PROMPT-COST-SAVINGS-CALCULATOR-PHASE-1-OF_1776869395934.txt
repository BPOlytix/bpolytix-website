TAB 1 — BOOKKEEPING & ACCOUNTING

User inputs:
- Role type (single select): Bookkeeper | Accountant | Hybrid finance support
- Number of entities (stepper +/-): default 1, min 1, max 10
- Monthly transactions (stepper +/-): default 100, increment 50, min 50, max 2000
- VAT returns required (toggle yes/no): default yes
- Payroll employees (stepper +/-): default 5, min 0, max 200
- Management accounts required (toggle yes/no): default no

In-house cost calculation (ZAR):

IF role = Bookkeeper:
  base_salary = 15698
IF role = Accountant:
  base_salary = 24748
IF role = Hybrid:
  base_salary = 20000

uif = base_salary * 0.01
sdl = base_salary * 0.01
seat = 4100
software = 1045
device = 800
recruitment = base_salary * 0.08
management = base_salary * 0.10

in_house_monthly = base_salary + uif + sdl + seat + software + device + recruitment + management

BPOLytix fee (ZAR):
IF monthly_transactions <= 150 AND management_accounts = no:
  bpolytix_fee = 8500   (Starter)
ELSE IF monthly_transactions <= 500 OR management_accounts = yes:
  bpolytix_fee = 14900  (Growth)
ELSE:
  bpolytix_fee = 21500  (Full)

IF role = Accountant: add R3,000 to bpolytix_fee
IF vat_returns = yes: add R1,200 to bpolytix_fee
IF payroll_employees > 10: add R800 to bpolytix_fee

saving_monthly = in_house_monthly - bpolytix_fee
saving_percent = (saving_monthly / in_house_monthly) * 100
saving_annual = saving_monthly * 12

Breakdown for donut chart (show where saving comes from):
- Salary burden saved: (uif + sdl + recruitment + management)
- Seat & infrastructure saved: (seat + software + device)
- Core salary differential: (base_salary - (bpolytix_fee * 0.7))
