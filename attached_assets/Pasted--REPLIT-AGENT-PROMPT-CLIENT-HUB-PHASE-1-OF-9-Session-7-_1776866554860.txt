ARCHITECTURE OVERVIEW:

Authentication:
- Supabase email+password auth only — no OAuth, no magic links
- Internal email format: username + "@bpolytix.internal"
  Example: username "jsmith" → Supabase auth email "jsmith@bpolytix.internal"
- Role stored in profiles table: 'admin' or 'client'
- After login: admin → /admin | client → /hub/dashboard

Database tables:
1. profiles — id, first_name, last_name, business_name, position, contact_number, address, username, role, is_active, created_at
2. projects — id, client_id, service_name, status, created_at
3. checklist_steps — id, project_id, step_number, step_label, status, updated_at
4. documents — id, project_id, uploaded_by, document_type, file_url, file_name, note, created_at

Checklist steps (universal, 8 steps):
1. Account activated
2. Welcome call scheduled
3. Contract signed
4. Project kick-off complete
5. Delivery in progress
6. Review & sign-off
7. Satisfied — invoice issued
8. Project complete / Ownership transferred

Step statuses: locked | in_progress | complete

Route structure:
/hub                         → login page (public)
/hub/dashboard               → client dashboard (protected — client only)
/hub/dashboard/[projectId]   → project detail (protected — client only)
/hub/profile                 → client profile (protected — client only)
/admin                       → admin overview (protected — admin only)
/admin/users                 → user list (protected — admin only)
/admin/users/new             → add user (protected — admin only)
/admin/users/[id]            → edit/delete user (protected — admin only)

Middleware rules:
- Unauthenticated user hits /hub/* → redirect to /hub
- Unauthenticated user hits /admin/* → redirect to /hub
- Authenticated client hits /admin/* → redirect to /hub/dashboard
- Authenticated admin hits /hub/dashboard → allow (admin can view client areas)

TASK 4 — Create middleware
Create artifacts/bpolytix/middleware.ts

Logic:
- Import createServerClient from @supabase/ssr
- On every request to /hub/* or /admin/*:
  1. Get the current session from Supabase
  2. If no session → redirect to /hub
  3. If session exists → fetch the user's role from the profiles table
  4. If role = 'client' and path starts with /admin → redirect to /hub/dashboard
  5. If role = 'admin' → allow all routes
  6. If role = 'client' → allow /hub/* routes only
- Export a config matcher:
  matcher: ['/hub/:path*', '/admin/:path*']
