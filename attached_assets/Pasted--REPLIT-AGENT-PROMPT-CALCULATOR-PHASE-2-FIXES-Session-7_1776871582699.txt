FIX 1 — Charts render for single service without clicking "Add another service"

In artifacts/bpolytix/app/calculator/page.tsx:

The ResultsCharts and ScreenshotCard components currently only render
when savedServices.length > 0. This means charts don't appear until
the user clicks "Add another service."

Fix: The charts must render as soon as the active tab has a valid
calculation result — not only when a service is saved.

Change the conditional from:
{savedServices.length > 0 && (
  <ResultsCharts ... />
  <ScreenshotCard ... />
)}

To a new condition that checks:
- Does the current active tab have a valid result? (inHouseMonthly > 0 or equivalent)
- OR does savedServices have items?

Specifically:
1. Compute activeResult by running the calculator function for the
   current active tab using current input state — this should already
   be happening for the metric tiles
2. Build a resultsToShow array:
   - Start with [...savedServices]
   - If activeResult is valid AND the active service is not already
     in savedServices, append activeResult to resultsToShow
3. Pass resultsToShow to ResultsCharts and ScreenshotCard
4. Only render charts if resultsToShow.length > 0 AND
   activeResult.savingMonthly > 0 (or savingTotal for project tabs)

This means: as soon as the page loads with default inputs,
the charts render immediately for Bookkeeping tab.
The user does not need to click anything to see the visuals.

FIX 2 — Gauge percentage text vertical alignment

In artifacts/bpolytix/components/calculator/ResultsCharts.tsx:

The RadialBarChart gauge uses startAngle=180 endAngle=0 — a half circle
opening downward. The center label is positioned using the chart's
full bounding box height, which places text too high.

Fix:
- The chart height is 160px
- The arc center is at y=0 (top of the half circle opens down)
- The visual center of the arc is at approximately y=10px from top
- Use a CustomLabel or absolute-positioned div overlay instead of
  Recharts label prop

Replace the existing center label with this approach:
- Wrap the RadialBarChart in a div with position: relative
- Set RadialBarChart height to 180px
- Place an absolutely positioned div:
  position: absolute
  top: 55%  (this places it in the visual center of the half-arc)
  left: 50%

- Inside this div:
  - Saving % value: Syne font, 42px, #00D4AA, fontWeight 700
  - "overall saving" label: DM Sans, 12px, #8892A4, marginTop: 2px

Remove any existing label or customLabel prop from the RadialBar
or RadialBarChart that was previously rendering the text.
